Kontekst
Aplikacja generuje wyjście w trybie output_both: wspólny arkusz + osobne pliki.
Po wygenerowaniu plików UI próbuje wczytać podgląd, ale rzuca:
FileNotFoundError: no such file: '.../output/.../billbo/billboard3_v_separated.pdf'
Widać, że ścieżka jest złamana („billbo” zamiast billboard3) – składanie ścieżki w UI jest błędne.
W trybie output_both miniatura ma zawsze pochodzić ze wspólnego arkusza (pierwsza strona), niezależnie od tego, że zapisano też osobne pliki.
Cel
W main_ui_home.py (metoda odpowiedzialna za budowę listy plików do podglądu — ta, w której log pokazał wyjątek wokół linii ~617) popraw:
Wyznaczanie ścieżki pliku do miniatury, gdy output_type == output_both.
Składanie ścieżek wyłącznie poprzez os.path.join/pathlib.Path, bez ręcznego „cięcia” lub wstawiania \.
Wybór prawidłowego pliku wspólnego arkusza w zależności od layoutu:
layout_vertical → szukaj *_v_separated.pdf
layout_horizontal → szukaj *_h_separated.pdf
Zastosuj fallback: jeśli znasz dokładną ścieżkę zwróconą przez moduł dzielący (np. splitter/SplitPDF) — użyj jej; jeśli nie — wyszukaj najnowszy pasujący plik w katalogu wyjściowym.
Zabezpiecz UI:
Sprawdź istnienie pliku (Path.exists()); jeśli nie istnieje, spróbuj:
(a) skorzystać z pełnej ścieżki zwróconej przez proces generowania (jeśli dostępna w stanie aplikacji),
(b) wyszukać globem w katalogu output_dir po wzorcu:
*_v_separated.pdf dla układu pionowego,
*_h_separated.pdf dla układu poziomego.
Gdy nic nie znaleziono, wyświetl komunikat LANG["msg_no_pdf_output"] i nie próbuj otwierać nieistniejącego pliku.
Nie mieszaj segmentów ścieżki z nazwą bazową:
Nazwę bazową weź z realnego wyjścia (np. z nazwy pliku wejściowego bez rozszerzenia albo wartości zwróconej przez SplitPDF).
Nigdy nie „przycinaj” jej (unikaj efektu billbo).
Wymagane zmiany (pseudo-kod)
W module main_ui_home.py, w miejscu gdzie budujesz files_to_preview dla podglądu:

from pathlib import Path
import glob

def _select_output_preview_file(self, output_dir: str, layout: str, output_type: str, last_result_paths: dict | None = None) -> Path | None:
    out_dir = Path(output_dir)

    # 1) jeśli mamy bezpośrednią ścieżkę wspólnego arkusza z procesu dzielenia – użyj jej
    if last_result_paths and "common_sheet_path" in last_result_paths:
        p = Path(last_result_paths["common_sheet_path"])
        if p.exists():
            return p

    # 2) fallback wg layoutu
    pattern = "*_v_separated.pdf" if layout == "layout_vertical" else "*_h_separated.pdf"

    # Szukaj tylko w bieżącym katalogu outputu (bez ręcznych sklejarek)
    candidates = sorted(out_dir.glob(pattern), key=lambda p: p.stat().st_mtime, reverse=True)
    return candidates[0] if candidates else None

def _update_output_preview(self):
    # ...
    layout = self.layout_var.get()                  # "layout_vertical" / "layout_horizontal"
    output_type = self.output_type_var.get()        # "output_both", "output_common_sheet", "output_separate_files"
    output_dir = self.output_dir_var.get()

    preview_file = None

    if output_type == "output_both":
        preview_file = self._select_output_preview_file(output_dir, layout, output_type, last_result_paths=self._last_result_paths)

    elif output_type == "output_common_sheet":
        preview_file = self._select_output_preview_file(output_dir, layout, output_type, last_result_paths=self._last_result_paths)

    elif output_type == "output_separate_files":
        # Dla osobnych plików: weź pierwszy wygenerowany bryt (najnowszy plik pasujący)
        # Np. "*_v.pdf" przy layout_vertical, "*_h.pdf" przy layout_horizontal
        suffix = "_v.pdf" if layout == "layout_vertical" else "_h.pdf"
        files = sorted(Path(output_dir).glob(f"*{suffix}"), key=lambda p: p.stat().st_mtime, reverse=True)
        preview_file = files[0] if files else None

    if not preview_file or not preview_file.exists():
        # elegancki fallback i komunikat
        self._show_output_preview_message(LANG["msg_no_pdf_output"])
        return

    # tu już zawsze mamy poprawną, istniejącą ścieżkę:
    doc = fitz.open(str(preview_file))  # bez string slicing; pełna ścieżka
    # ... reszta renderu miniatury bez zmian


Uwaga: self._last_result_paths to słownik, który proszę uzupełnić podczas generowania PDF (w miejscu, gdzie już logujesz „Zapisano wspólny arkusz: …”). Przekaż tam pełne ścieżki np.:

self._last_result_paths = {
    "common_sheet_path": full_common_sheet_path,   # Path/str
    "separate_files_paths": list_of_slice_paths,   # list[Path/str]
}
Dzięki temu UI nie musi zgadywać nazwy pliku.

Zero „ręcznego” sklejania backslashy, zero skracania nazw plików.

Zabezpiecz wyjątek fitz.open i pokaż przyjazny komunikat zamiast crasha.
